# Contributing to Matrice AI

Thank you for your interest in contributing to Matrice AI\!

## Code of Conduct

Please be respectful and professional in all interactions.

## Getting Started

1. **Fork/Clone** the repository you want to contribute to
2. **Create a branch** from `dev` with a descriptive name
3. **Make your changes** following our coding standards
4. **Test thoroughly** before submitting
5. **Submit a PR** with a clear description

## Branch Naming Convention

Use the following format for branch names:
- `feature/<description>` - New features
- `bugfix/<description>` - Bug fixes
- `hotfix/<description>` - Urgent production fixes
- `refactor/<description>` - Code refactoring
- `docs/<description>` - Documentation updates

Example: `feature/add-user-authentication`

## Commit Messages

Write clear, concise commit messages:
- Use present tense ("Add feature" not "Added feature")
- Use imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit first line to 72 characters
- Reference issues when applicable

Example:
```
Add user authentication endpoint

- Implement JWT token generation
- Add login/logout routes
- Include unit tests

Closes #123
```

## Pull Request Process

1. Update documentation if needed
2. Add/update tests for your changes
3. Ensure all tests pass
4. Request review from code owners
5. Address review feedback promptly
6. Squash commits if requested

## Code Review Guidelines

- Be constructive and respectful
- Focus on the code, not the person
- Explain the "why" behind suggestions
- Approve when satisfied, not perfect

## Questions?

Contact your team lead or reach out in the team channel.
